# AI量化交易平台 - 常见问题 FAQ

## 一、环境安装

### Q1: 依赖安装失败 (找不到合适的版本)

**问题**: 安装pandas/numpy等包时报错 `ERROR: Could not find a version that satisfies the requirement`

**原因**: Windows 32位Python可能缺少预编译wheel包。

**解决方案**:
```powershell
# 使用国内镜像源
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 如果还是失败，指定具体版本
pip install pandas==2.0.3 numpy==1.26.4
```

### Q2: 编译错误 (C++编译器问题)

**问题**: 安装时出现 `error: Microsoft Visual C++ 14.0 or greater is required`

**原因**: 部分包需要从源码编译，Windows缺少C++编译器。

**解决方案**:
```powershell
# 方案1: 安装Visual Studio Build Tools
# 下载地址: https://visualstudio.microsoft.com/visual-cpp-build-tools/
# 安装时勾选 "C++ build tools"

# 方案2: 使用预编译wheel (推荐)
pip install --only-binary :all: <包名>

# 方案3: 使用conda (如果安装了miniconda/anaconda)
conda install scikit-learn xgboost lightgbm
```

### Q3: AKShare导入报错

**问题**: `ModuleNotFoundError: No module named 'curl_cffi'` 或 `No module named 'py_mini_racer'`

**解决方案**:
```powershell
pip install py_mini_racer
pip install curl_cffi --prefer-binary
```

### Q4: 虚拟环境激活失败

**问题**: Windows PowerShell执行策略限制

**解决方案**:
```powershell
# 以管理员身份运行PowerShell，执行:
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

# 然后再激活:
.\.venv\Scripts\activate
```

### Q6: 获取历史数据失败 (东方财富连接被断开)

**问题**: `get_stock_history("000001")` 报错 `Remote end closed connection without response`

**原因**: 东方财富API反爬限制，非中国网络环境或高频请求会被断开连接。

**解决方案** (v0.1.1已自动处理):
```python

# 采集器已默认使用腾讯证券数据源，无需手动切换
collector = DataCollector()
df = collector.get_stock_history("000001")  # 自动走腾讯证券接口

# 如果需要手动指定数据源，可以直接调用AKShare
import akshare as ak

# 腾讯证券 (推荐，稳定)
df = ak.stock_zh_a_hist_tx(symbol="sz000001", adjust="qfq")

# 新浪财经 (备选)
df = ak.stock_zh_a_daily(symbol="sh000001", adjust="qfq")
```

## 三、回测使用

### Q8: 回测结果不准确

**问题**: 回测收益率与实际不符

**常见原因**:
- 未考虑手续费和滑点
- 信号计算使用了未来数据
- 委托成交逻辑过于理想

**解决方案**:
```python
# 设置合理的交易成本
engine = BacktestEngine(
    initial_cash=100000,
    commission=0.0003,    # 万三手续费
    slippage=0.0001,      # 万一滑点
    stamp_duty=0.001,     # 千一印花税(卖出)
)

# 检查策略中是否使用了未来数据
# 错误: 使用当前bar的close计算信号，实际交易时只有前一个bar的数据可用
# 正确: 使用前一个bar的数据计算信号
```

### Q9: 策略信号一直为HOLD

**问题**: 策略所有bar都返回 `Signal.HOLD`

**可能原因**:
- 指标计算需要足够的数据点
- 条件判断逻辑错误
- 数据处理有NaN值

**调试方法**:
```python
# 添加调试日志
def next(self, bar):
    idx = self.data.index.get_loc(bar.name)
    print(f"Bar {idx}: close={bar['close']}, volume={bar['volume']}")
    
    # 检查指标值
    if idx >= self.slow_period:
        print(f"  fast_sma={self.data['sma_fast'].iloc[idx]:.2f}, slow_sma={self.data['sma_slow'].iloc[idx]:.2f}")
```

### Q10: 回测速度很慢

**问题**: 大量数据回测耗时过长

**解决方案**:
```python
# 1. 使用向量化计算替代循环
# 2. 减少日志输出
# 3. 只回测关键时间段
df = df.loc["2022-01-01":"2023-12-31"]  # 缩短回测区间

# 4. 使用日线数据替代分钟数据
```

### Q11: 如何添加新策略？

**步骤**:
```python
# 1. 在 src/strategy/ 下创建新文件
# 2. 继承 BaseStrategy 类
from src.strategy.base import BaseStrategy, Signal

class MyStrategy(BaseStrategy):
    def init(self):
        """初始化"""
        pass
    
    def next(self, bar) -> Signal:
        """生成信号"""
        return Signal.HOLD

# 3. 在 __init__.py 中注册
# 4. 在 scripts/run_backtest.py 中调用
```

### Q13: 参数优化如何做？

```python
# 简单的网格搜索
from src.backtest.engine import BacktestEngine
from src.strategy.trend_strategy import SmaCrossStrategy

best_params = None
best_return = -float('inf')

for fast in [5, 10, 20]:
    for slow in [20, 30, 60]:
        if fast >= slow:
            continue
        engine = BacktestEngine(initial_cash=100000)
        strategy = SmaCrossStrategy(params={'fast_period': fast, 'slow_period': slow})
        result = engine.run(data, strategy)
        
        if result.total_return > best_return:
            best_return = result.total_return
            best_params = {'fast': fast, 'slow': slow}

print(f"最优参数: {best_params}, 收益率: {best_return:.2%}")
```
