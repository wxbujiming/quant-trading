## 一、项目概述

### 1.1 项目背景

本项目旨在构建一个个人使用的AI量化交易平台，整合以下开源项目的优点：
- **Qlib** (微软): AI因子挖掘、模型训练
- **VeighNa** (vn.py): 实盘交易、券商接口
- **Backtrader**: 回测框架设计思路

### 1.2 目标市场

| 市场 | 交易频率 | 策略类型 | 优先级 | 当前状态 |
|------|----------|----------|--------|----------|
| A股 | 日线级别 | 技术指标策略、因子选股、AI预测 | ⭐⭐⭐⭐⭐ | ✅ 回测完成 |
| 期货 | 日线/小时级 | CTA趋势、ATR止损、套利 | ⭐⭐⭐⭐⭐ | 🔄 期货CTA开发中 |
| 美股 | 日线/高频 | 趋势跟踪 | ⭐⭐⭐ | ⬜ 待开始 |

### 1.3 资源约束

- **开发环境**: 个人电脑 (本地开发、AI训练、回测)
- **服务器**: 阿里云 2C2G (数据采集、API服务)
- **部署方式**: 本地部署为主

    """期货回测引擎: 保证金/多空双向/T+0/逐日盯市"""

    def close_long(self, symbol, price, volume)   # 平多

## 六、期货CTA系统

> 参见完整实现：`src/backtest/futures_engine.py` | `src/strategy/futures_strategy.py`

### 6.1 期货回测引擎核心特色

| 特性 | 说明 |
|------|------|
| **保证金交易** | 只需占用保证金比例，支持杠杆 |
| **多空双向** | 既可做多也可做空 |
| **T+0** | 当天开仓可当天平仓 |
| **逐日盯市** | 每日按结算价计算浮动盈亏 |
| **合约乘数** | 各品种价格×乘数=合约价值 |
| **平今手续费** | 平今仓手续费单独计算 |
| **ATR止损** | 基于真实波幅的动态止损 |
| **移动止损** | 盈利后自动上移止损线 |

### 6.2 支持品种及合约参数

| 品种 | 代码 | 乘数 | 保证金率 | 最小变动 | 交易时间 |
|------|------|------|----------|----------|----------|
| 沪铜 | CU | 5吨 | 12% | 10元/吨 | 日盘+夜盘 |
| 棕榈油 | P | 10吨 | 10% | 2元/吨 | 日盘+夜盘 |
| 螺纹钢 | RB | 10吨 | 10% | 1元/吨 | 日盘+夜盘 |
| 原油 | SC | 1000桶 | 15% | 0.1元/桶 | 日盘+夜盘 |
| 沪深300股指 | IF | 300元/点 | 12% | 0.2点 | 日盘 |

### 6.3 CTA策略核心逻辑

#### 双均线CTA策略 (DualMaCrossStrategy)

```
参数: fast_period=10, slow_period=30, atr_multiple=2.0
      risk_per_trade=0.02, use_trailing_stop=True

逻辑:
  1. 计算快速MA和慢速MA
  2. 金叉(快上穿慢): 开多
  3. 死叉(快下穿慢): 开空
  4. 止损: ATR × atr_multiple (动态调整)
  5. 移动止损: 盈利超过ATR×1.5后启动
  6. 仓位管理: 基于 risk_per_trade (凯利风格)
```

#### 趋势通道策略 (SimpleTrendStrategy)

```
参数: entry_period=20, exit_period=20
      atr_period=14, atr_multiple=2.0

逻辑:
  1. 计算唐奇安通道 (20日最高/最低)
  2. 突破上轨: 开多
  3. 突破下轨: 开空
  4. 跌破中轨(均值): 平多
  5. 涨破中轨(均值): 平空
  6. ATR止损保护
```

## 九、风险控制体系

### 9.1 期货特有风险

| 风险类型 | 说明 | 应对措施 |
|----------|------|----------|
| **保证金风险** | 持仓亏损导致保证金不足 | 实时风险度监控, 风险度>80%预警 |
| **隔夜跳空** | 次日开盘跳空导致大幅亏损 | 控制隔夜仓位, 设置止损单 |
| **流动性风险** | 非主力合约成交稀疏 | 只交易主力合约, 持仓量过滤 |
| **交割风险** | 进入交割月被强平 | 提前换月, 避免进入交割月 |
| **强平风险** | 风险度>100%被期货公司强平 | 提前减仓/追加保证金 |

### 9.2 风控规则清单

| 规则 | 级别 | 说明 |
|------|------|------|
| 单品种最大持仓 | 硬性 | 不超过总资金20% |
| 总仓位比例 | 硬性 | 不超过总资金50% |
| 每日最大亏损 | 硬性 | 当日亏损超5%暂停交易 |
| 通用止损线 | 硬性 | 单笔亏损超总资金2% |
| 交易频率限制 | 软性 | 同品种同一方向2秒内不重复 |
| 风险度监控 | 硬性 | 风险度>85%告警, >95%减仓, >100%强平 |
| 自成交防范 | 硬性 | 同一账户不可自买自卖 |
| 涨跌停板保护 | 硬性 | 涨停不开空/跌停不开多 |
| 交割月限制 | 硬性 | 交割月前一个月停止开新仓 |
| 撤单频率限制 | 硬性 | 频繁撤单会被交易所限制 |

### 9.3 止损体系

```
第一层: 策略止损 (ATR止损 / 固定止损)
  └── 每笔交易自带止损线
  └── 盈利后自动上移 (移动止损)

第二层: 账户止损 (每日亏损限制)
  └── 当日亏损 > 总资金5% → 停止当日交易
  └── 月度亏损 > 总资金15% → 停止当月交易

第三层: 系统止损 (风险度强制干预)
  └── 风险度 > 85% → 告警通知
  └── 风险度 > 95% → 自动减仓50%
  └── 风险度 > 100% → 全部平仓
```

### 9.4 保证金计算

```
保证金 = 合约价值 × 保证金率
合约价值 = 价格 × 合约乘数 × 手数

示例 (沪铜CU, 70000元/吨):
  1手合约价值 = 70000 × 5 = 350,000元
  交易所保证金 = 350,000 × 12% = 42,000元
  期货公司保证金 = 350,000 × 14% = 49,000元 (通常略高于交易所)
  
风险度 = 占用保证金 / 总权益 × 100%
  - 风险度 > 100%: 保证金不足, 面临强平
  - 安全线: 风险度 < 60%
  - 预警线: 风险度 > 80%
  - 危险线: 风险度 > 95%
```

### 10.5 Web监控面板 ✅

Streamlit 多页面监控面板，支持双数据源模式：

**数据源：**
- **实时模式**: 引擎同一进程内直接读取各管理器状态
- **快照模式**: 引擎未运行时读取 `data/live_state/` 下 JSON 快照文件
- **跨进程命令**: 通过 `data/live_state/commands.json` 文件通道，面板向另一进程的引擎发送暂停/恢复/停止/全平/撤单命令

**页面列表：**

| 页面 | 功能 |
|------|------|
| 总览 | 账户指标(总权益/可用/盈亏/持仓数) + 风险度 + 今日告警 |
| 持仓 | 持仓表格(合约/方向/手数/均价/盈亏) |
| PnL曲线 | Plotly交互权益曲线 + 最大回撤等指标 |
| 订单 | 活跃订单 + 历史成交 |
| 日志 | 实时日志查看(级别颜色高亮) |
| 风控 | 风险度仪表盘 + 告警列表 + 自动减仓预案 |
| 控制 | 暂停/恢复/停止/全平/撤单按钮(带二次确认) |

**启动方式：**
```bash
# 启动引擎（另一终端）
.venv/Scripts/python scripts/run_live_engine.py

# 启动监控面板
.venv/Scripts/python -m streamlit run web/app.py
```

## 十三、安全考虑

### 13.1 数据安全

- 敏感配置不提交Git (.gitignore)

- 交易记录不可篡改 (追加日志)

- 操作日志记录 (谁/什么时间/做了什么)
- 权限控制 (API密钥权限最小化)

### 13.3 网络安全

- CTP使用TLS加密传输
- API密钥不硬编码在代码中
- 使用专用交易网络 (可考虑独立网卡)
